The Template Method pattern is implemented in the Report abstract class through the generateReport() method, which is declared final to lock the workflow order: loadData, formatHeader, formatBody, and formatFooter. Each step is defined as a protected abstract method that subclasses must override to supply their own content. StudentReport and CourseReport each implement the variable steps without modifying the algorithm's structure. The driver class uses List<Report> and polymorphism so generateReport() dispatches to the correct subclass at runtime, demonstrating how the pattern separates the invariant workflow from the variant steps.
